# LandScope — 토지 실거래 조회 · PF 간이 사업성 분석

국토교통부 공공데이터를 수집·정제해 **토지 매매 실거래를 권역별로 조회**하고, 선택한 필지의 실거래가를 바탕으로 **PF 초기 검토용 약식 사업수지**를 즉시 계산하는 웹 서비스입니다.

부동산 대체투자(PF) 실무에서 개발사업 초기 검토는 "토지 시세 확인 → 용도지역·개발조건 파악 → 약식 수지 검토" 순으로 진행됩니다. 이 흐름을 그대로 웹 서비스로 구현했습니다.

## 주요 기능

### 1. 토지 실거래 통합 조회 (`/explorer`)
- 대상 권역: 99개 시군구 (서울 25 · 인천 11 · 경기 47 · 충남 16) — [src/lib/config.ts](src/lib/config.ts)
- 법정동 코드는 실거래가 API에 전수 조회해 검증 (화성시 4구 신설, 부천시 3구 재설치, 인천 2026-07 개편 등 행정구역 변경 반영)
- 계약기간·면적(평)·지목 필터, 평당가 자동 계산
- 카카오맵 위에 **읍면동 단위 집계**(거래 건수·평균 평당가) 오버레이 표시, 클릭 시 해당 동으로 리스트 필터링
- 필지 선택 → "이 필지로 사업성 분석" 버튼으로 실거래가가 계산기에 자동 입력

### 2. 약식 사업수지 분석 (`/calculator`)
PF 심사 초기의 약식 사업수지표를 웹으로 구현한 것으로, 이 서비스의 핵심 기능입니다.

- **개발 상품 유형**(아파트/오피스텔/지식산업센터/물류센터/타운하우스/근생) 선택 시 공사비·분양비율·판매비율·사업기간 기준값 자동 적용 + 용도지역 입지 불일치 경고
- **용도지역 선택 시 국토계획법 시행령상 건폐율·용적률 상한 자동 적용** (조례 하향 가능성 안내 포함)
- 원가 세분화: 토지비, 취득부대비(취득세 4.6%), 공사비, 설계·감리비, 예비비, 판매비(분양수입 대비), 이자비용(평균 인출률 반영), PF 취급수수료
- 산출: 항목별 수지표, 개발이익(세전), **사업이익률**, **자기자본수익률·연환산 ROE**
- **분양가 × 공사비 민감도 매트릭스** (±10%, 5×5) — 이익률 구간별 색상 표시
- **엑셀 모델 다운로드**: 값이 아닌 **수식으로 작성된 워크북** — 가정 셀(B열)을 수정하면 산출부와 민감도표가 함께 재계산됨 (exceljs)

계산 로직: [src/lib/feasibility.ts](src/lib/feasibility.ts) · 엑셀 생성: [src/lib/excel.ts](src/lib/excel.ts)

```
연면적 = 대지면적 × 용적률,  분양면적 = 연면적 × 분양가능면적비율
분양수입 = 분양면적 × 분양단가 × 분양률
직접사업비 = 토지비 + 취득부대비 + 공사비 + 설계감리비 + 예비비
판매비 = 분양수입 × 판매비율
대출원금 = 직접사업비 × LTC
금융비용 = 원금 × 평균인출률 × 금리 × 기간(년) + 원금 × 취급수수료율
총사업비 = 직접사업비 + 판매비 + 금융비용
사업이익률 = 이익/총사업비,  ROE = 이익/자기자본,  연환산 ROE = ROE ÷ 기간(년)
```

## 아키텍처

```
[국토교통부 실거래가 API]
        │  (Vercel Cron 매일 06:00 KST, Bearer 인증)
        ▼
[/api/ingest]  XML 파싱 → 정규화 → Supabase upsert (자연키 중복 방지)
        ▼
[Supabase Postgres: land_transactions]  ← RLS로 읽기만 공개
        ▲
[/api/land]  기간/면적/금액/지목 필터 조회
        ▲
[Next.js 프론트]  /explorer (지도+리스트) · /calculator (순수 클라이언트 계산)
```

- **크롤링 없음** — 공공데이터포털 공식 API만 사용 (법적 리스크 없음)
- 서비스키·service role 키는 서버 사이드에서만 사용, 클라이언트에는 미노출
- 실거래 데이터는 DB에 적재해 서빙 (매 요청마다 공공 API를 호출하지 않음 — 응답속도·쿼터 관리)

## 기술 스택

Next.js 16 (App Router) · TypeScript · Tailwind CSS 4 · Supabase (Postgres) · Kakao Maps SDK · Vercel (호스팅+Cron)

## 실행 방법

1. **의존성 설치**: `npm install`
2. **키 발급**
   - [공공데이터포털](https://www.data.go.kr/data/15126466/openapi.do)에서 "토지 매매 실거래가" 활용신청 → 일반 인증키(Decoding)
   - [Supabase](https://supabase.com) 프로젝트 생성 → SQL Editor에서 [supabase/schema.sql](supabase/schema.sql) 실행
   - [Kakao Developers](https://developers.kakao.com)에서 앱 생성 → JavaScript 키 (플랫폼에 도메인 등록)
3. **환경변수**: `.env.local.example`을 `.env.local`로 복사해 값 입력
4. **데이터 적재**: `npm run dev` 후
   ```
   curl -H "Authorization: Bearer <INGEST_SECRET>" "http://localhost:3000/api/ingest?months=6"
   ```
5. http://localhost:3000 접속

## 배포 (Vercel)

1. GitHub 저장소 연결 후 Vercel 프로젝트 생성
2. 환경변수 5개 등록 (`.env.local.example` 참고)
3. `vercel.json`의 Cron이 매일 21:00 UTC(06:00 KST)에 최근 2개월치 데이터를 자동 갱신
   - Vercel Cron은 Authorization 헤더를 직접 못 붙이므로, 프로젝트 설정의 `CRON_SECRET` 방식을 쓰거나 ingest 라우트의 인증 방식을 환경에 맞게 조정
4. Kakao Developers에 배포 도메인 추가

## 한계와 향후 개선

- **지번 일부 비공개**: 실거래가 API가 개인정보보호를 위해 지번을 일부만 제공 → 필지 단위 정밀 매칭(PNU) 불가. 개별공시지가·토지이용계획 API 연계는 필지 특정이 가능한 범위에서 단계적으로 추가 예정
- **수지 모델 단순화**: 분양률 100% 가정, 금융비용 전액·전기간 부과(보수적), 취득세·부가세 미분리. 실무 수지표 대비 항목이 간소함 — 시나리오 비교(분양가 ±10%), 민감도 분석이 다음 단계
- **용도지역 상한은 법령 기준**: 실제 적용치는 지자체 조례로 하향될 수 있어 참고값으로 안내
- 읍면동 지오코딩은 클라이언트에서 카카오 geocoder 사용 — 거래량이 많아지면 서버 사이드 캐싱으로 전환
